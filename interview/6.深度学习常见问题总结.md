# 常见问题总结

## 1. 梯度消失、爆炸原因及其解决方法

**原因: **

- 发生在深层的神经网络中
- 反向传播

**方案**

- 1. 方案1-预训练加微调
  
每次训练一层隐节点，训练时将上一层隐节点的输出作为输入，而本层隐节点的输出作为下一层隐节点的输入

- 2. 方案2-梯度剪切、正则

梯度剪切这个方案主要是针对梯度爆炸提出的，其思想是设置一个梯度剪切阈值，然后更新梯度的时候，如果梯度超过这个阈值，那么就将其强制限制在这个范围之内。这可以防止梯度爆炸。

另外一种解决梯度爆炸的手段是采用权重正则化（weithts regularization）比较常见的是l1正则，和l2正则

- 3. 方案3-relu、leakrelu、elu等激活函数

relu的优缺点: 

```
1. 解决了梯度消失、爆炸的问题
2. 计算方便，计算速度快
3. 加速了网络的训练

缺点: 
1. 由于负数部分恒为0，会导致一些神经元无法激活（可通过设置小学习率部分解决）
2. 输出不是以0为中心的
```

- 4. 解决方案4-batchnorm
- 5. 解决方案5-残差结构
- 6. 解决方案6-LSTM替代rnn

## 2. 过拟合原因和解决方法

过拟合（overfitting）现象的发生——神经网络模型在训练集上表现很好，但对新数据预测时效果不好

**如何知道模型产生过拟合**

将数据划分为训练集、验证集、测试集，训练集效果好，测试集差

**解决过拟合**

- 0. 更多的数组
- 1. 正则化
- 2. dropout
- 3. 提取停止: 最后的模型不一定是最好的
- 4. 集成学习方法
- 5. batch normalization

## 3. dropout的实现细节

dropout 有两种实现方式，Vanilla Dropout 和 Inverted Dropout

输入: $x = [x_1, x_2, x_3, x_4]$, 正常的回溯梯度为: $d = [d_1, d_2, d_3, d_4]$, 假设以p=0.2进行伯努利采用得到$mask = [1, 0, 1, 1]$

Inverted把所有的修改都放到了训练阶段，保持预测阶段不变，在forward的时候要进行放大1/(1-p), $x' = 1.25*[x_1, 0, x_3, x_4]$。在backward更新梯度的时候同样进行放大1/(1-p), $d' = 1.25*[d_1, 0, d_3, d_4]

**为什么需要rescale**

为了让在infer的时候不进行dropout, 保持分布一致性。
